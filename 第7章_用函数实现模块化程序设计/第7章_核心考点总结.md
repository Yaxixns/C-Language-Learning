# 第7章 用函数实现模块化程序设计 — 核心考点总结

---

## 一、本章核心知识点

### 1.1 函数定义与调用（⭐必考）

```c
// 函数定义
返回值类型 函数名(形参列表)
{
    函数体;
    return 表达式;    // 若无返回值，写 return; 或省略
}

// 函数声明（放在调用之前或头文件中）
返回值类型 函数名(形参列表);

// 函数调用
函数名(实参列表);
```

| 要点 | 说明 |
|------|------|
| 返回值类型 | 不能是数组，可以是 int/float/double/char/void/指针 |
| void 函数 | 无返回值，return 可有可无 |
| 形参 vs 实参 | **单向值传递**，形参改变不影响实参（除非传地址） |
| 声明与定义 | 声明是告诉编译器函数存在，定义是实现函数体 |

### 1.2 函数参数传递（⭐高频）

#### 1.2.1 值传递 vs 地址传递

```c
// 值传递（传变量）：修改形参不影响实参
void swap(int a, int b) { int t=a; a=b; b=t; }  // 无效！

// 传数组：数组名就是首地址，函数内修改影响实参
void sort(int arr[], int n) { ... }  // arr 的元素被修改
```

#### 1.2.2 数组作为函数参数 — 四种等价写法（⭐必考）

这四种写法**完全等价**，编译器都当成 `int *` 来处理：

```c
// 形式1：带方括号（最常用）
void f(int a[], int n)     { ... }

// 形式2：指定长度（写了也白写，编译器会忽略这个数字）
void f(int a[10], int n)   { ... }

// 形式3：指针（等价于形式1）
void f(int *a, int n)      { ... }

// 形式4：多维数组的情况
void f(int a[][10], int n) { ... }
```

> ⚠️ **关键点**：数组形参本质是指针，`sizeof(a)` 返回的是指针大小（4或8字节），**不是**数组大小！所以数组长度必须单独传（`int n`）。

| 写法 | 含义 | 备注 |
|------|------|------|
| `void f(int a[])` | 一维数组 | 等价于 `int *a` |
| `void f(int *a)` | 指针 | 跟上面完全一样 |
| `void f(int a[10])` | 写了长度 | 编译器**忽略**，还是指针 |
| `void f(int a[][5])` | 二维，列数必须写 | 等价于 `int (*a)[5]` |

#### 1.2.3 字符串作为函数参数（⭐高频）

字符串本质上就是字符数组，传参规则完全一样：

```c
// 方式1：char 数组形式（最常用）
void reverse(char s[])
{
    // s 就是指向实参的指针
    s[0] = 'X';    // 能修改实参内容
}

// 方式2：char 指针形式（等价）
void reverse(char *s)
{
    *s = 'X';      // 效果同上
}

// 方式3：const 保护（不修改原串时用）
void print(const char s[])
{
    // s[0] = 'X';  ← 编译报错，受 const 保护
}
```

#### 1.2.4 调用时的几种传参方式

```c
void f(char s[])  { ... }

int main()
{
    char s1[100] = "Hello";     // 字符数组
    char *s2 = "World";         // 字符指针
    char s3[] = "abc";          // 不指定长度

    f(s1);       // ✅ 传数组名
    f(s2);       // ✅ 传指针
    f(s3);       // ✅ 传数组名
    f("xyz");    // ✅ 直接传字符串常量（但函数内不可修改）
    f(&s1[0]);   // ✅ 传首元素地址

    return 0;
}
```

> 💡 总结：只要传的是 `char*`（地址），怎么传都行。数组名、指针、字符串常量、`&s[0]` 都能接。

#### 1.2.5 一图总结：数组/字符串传参全形式

```
                ┌─────────────────────────────┐
                │      函数定义端（形参）        │
                ├─────────────────────────────┤
                │ void f(char s[])             │  ← 最常用
                │ void f(char s[100])          │  ← 长度写了白写
                │ void f(char *s)              │  ← 指针形式等价
                │ void f(const char s[])       │  ← 只读保护
                └─────────────────────────────┘
                              ↕  完全等价
                ┌─────────────────────────────┐
                │      函数调用端（实参）        │
                ├─────────────────────────────┤
                │ f(arr)     // 传数组名        │
                │ f(ptr)     // 传指针变量       │
                │ f("abc")   // 传字符串常量     │
                │ f(&arr[0]) // 传首元素地址     │
                └─────────────────────────────┘
```

### 1.3 递归函数（⭐难点）

```c
// 递归必须有两个要素：
// ① 递归公式（递推关系）
// ② 递归终止条件（边界）
int fact(int n)
{
    if (n == 1) return 1;           // 终止条件
    else return n * fact(n - 1);    // 递归公式
}
```

| 经典递归 | 公式 |
|------|------|
| 阶乘 | n! = n × (n-1)!，1! = 1 |
| 斐波那契 | F(n) = F(n-1) + F(n-2)，F(1)=F(2)=1 |
| 勒让德多项式 | \( p_n(x) = \frac{(2n-1)x·p_{n-1}(x) - (n-1)p_{n-2}(x)}{n} \)，p₀=1, p₁=x |

### 1.4 变量的作用域与存储类别（⭐常考）

#### 局部变量 vs 全局变量
| | 局部变量 | 全局变量 |
|------|------|------|
| 定义位置 | 函数内部 | 所有函数外 |
| 作用域 | 本函数内 | 整个源文件 |
| 生命期 | 函数执行期间 | 程序运行期间 |
| 默认值 | 未初始化，值随机 | 自动初始化为0 |

#### 存储类别
| 关键字 | 生命期 | 作用域 |
|------|------|------|
| `auto` | 局部（默认） | 本函数 |
| `static` 局部 | 全局（只初始化一次） | 本函数 |
| `extern` | 全局 | 跨文件 |
| `register` | 局部 | 本函数（建议放寄存器） |

## 二、高频考点（期末必考）

### 🔥 考点1：值传递 vs 地址传递
- 传普通变量 → 不能修改实参
- 传数组 → 能修改实参元素

### 🔥 考点2：递归程序阅读
- 给递归函数，写输出结果
- **关键**：画出递归调用树，从终止条件往回推

### 🔥 考点3：static 局部变量
```c
void f() { static int a = 0; a++; printf("%d", a); }
// 调用 n 次，输出 1,2,3...（不会每次都初始化为0）
```

### 🔥 考点4：模块化编程
- 函数功能单一、高内聚低耦合
- 先声明后调用

## 三、易错点警示（⚠️ 坑点汇总）

| 易错点 | 说明 |
|--------|------|
| 值传递无法交换 | `swap(a,b)` 传值无法真正交换 |
| 递归无终止条件 | 导致栈溢出（stack overflow） |
| static 局部变量 | 只初始化一次，再次调用时保留上次的值 |
| 函数声明遗漏 | 函数定义在调用之后时必须提前声明 |
| 数组形参 | `void f(int a[])` 和 `void f(int *a)` 等价 |
| 全局变量与局部变量同名 | 局部覆盖全局 |
