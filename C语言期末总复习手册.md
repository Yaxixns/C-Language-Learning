# 第1章 程序设计和C语言

## 1.1 程序与程序设计

| 概念             | 定义                                             |
| ---------------- | ------------------------------------------------ |
| **程序**         | 计算机能够识别和执行的一组指令（有序的指令集合） |
| **程序设计**     | 设计、编制、调试程序的过程                       |
| **程序设计语言** | 人与计算机交流的规则化语言                       |

## 1.2 计算机语言的发展

```
机器语言 → 汇编语言 → 高级语言
（二进制）  （助记符）   （接近自然语言）
```

- **机器语言**：计算机唯一可直接执行的语言，但难以记忆和使用
- **汇编语言**：用助记符代替二进制，需汇编程序翻译
- **高级语言**：接近人类自然语言和数学语言，需编译程序翻译，可移植性好

## 1.3 C语言的发展与特点

- **诞生**：1972年，贝尔实验室，Dennis Ritchie 设计
- **ANSI C 标准**：1983年制定，1989年通过（ANSI X3.159-1989）
- **ISO C 标准**：1990年，国际标准化组织采纳（ISO/IEC 9899:1990）
- **C99/C11/C17**：后续标准更新

**C语言的主要特点**（⭐常考简答）：

1. 语言简洁、紧凑，使用方便灵活（仅37个关键字）
2. 运算符丰富（34种运算符）
3. 数据类型丰富（整型、浮点型、字符型、指针、结构体等）
4. 具有结构化的控制语句
5. 语法限制不太严格，程序设计自由度大
6. 允许直接访问物理地址，能进行位操作
7. 生成目标代码质量高，程序执行效率高
8. 可移植性好

## 1.4 简单C语言程序的结构（⭐核心必考）

```c
#include <stdio.h>           // 编译预处理指令（头文件）

int main()                   // 主函数（程序入口）
{
    printf("Hello World\n"); // 函数体（语句）
    return 0;                // 返回值
}
```

**程序结构要点**：

1. 一个C程序**有且仅有一个** `main` 函数
2. 程序从 `main` 函数开始执行，`main` 函数执行完毕程序结束
3. `#include` 是**编译预处理指令**，不是语句（不用加分号）
4. 每个语句以 `;` 结束
5. `/* ... */` 为块注释，`//` 为行注释
6. C语言本身不提供输入输出语句，由库函数（如 `printf`、`scanf`）实现

## 1.5 C程序的开发流程（⭐常考填空）

```
编辑(.c源文件) → 编译(.obj目标文件) → 连接(.exe可执行文件) → 运行
```

| 阶段 | 产物              | 说明                         |
| ---- | ----------------- | ---------------------------- |
| 编辑 | `.c` 源文件       | 编写C源代码                  |
| 编译 | `.obj` 目标文件   | 编译器将源程序翻译成机器指令 |
| 连接 | `.exe` 可执行文件 | 将目标文件与库函数等连接装配 |
| 运行 | 输出结果          | 执行程序                     |

# 第2章 算法——程序的灵魂

## 2.1 算法的概念（⭐必考）

| 概念     | 定义                                           |
| -------- | ---------------------------------------------- |
| **算法** | 为解决一个问题而采取的方法和步骤，是程序的灵魂 |
| **程序** | 数据结构 + 算法 + 程序设计方法 + 语言工具      |

**通俗理解**：算法 = 做事的步骤。比如菜谱、洗衣机流程、乐谱等。

## 2.2 算法的五大特性（⭐高频简答）

1. **有穷性**：有限步骤后结束，每一步在有限时间内完成
2. **确定性**：每一步都有确切的含义，不能有二义性
3. **有零个或多个输入**：从外界获取必要信息
4. **有一个或多个输出**：必须产生结果
5. **有效性**：每一步都能被有效执行，能得到确定结果

## 2.3 算法的表示方法（⭐常考选择题）

| 方法           | 说明                   | 特点                   |
| -------------- | ---------------------- | ---------------------- |
| 自然语言       | 用日常语言描述         | 通俗但易歧义           |
| **传统流程图** | 用图形符号表示算法     | 直观形象，但占用篇幅大 |
| **N-S 流程图** | 盒图，去掉流程线       | 结构化，紧凑           |
| **伪代码**     | 介于自然语言和代码之间 | 灵活，接近程序         |
| 计算机语言     | 用 C 等语言实现        | 计算机可直接执行       |

**流程图常用符号**：

| 符号       | 名称        | 含义          |
| ---------- | ----------- | ------------- |
| 圆角矩形   | 起止框      | 算法开始/结束 |
| 矩形       | 处理框      | 计算/赋值     |
| 菱形       | 判断框      | 条件判断      |
| 箭头       | 流程线      | 执行方向      |
| 平行四边形 | 输入/输出框 | 数据读写      |

## 2.4 三种基本结构（⭐核心必考）

| 结构         | 特点                               |
| ------------ | ---------------------------------- |
| **顺序结构** | 按先后顺序依次执行，无分支、无跳转 |
| **选择结构** | 根据条件判断，选择执行某个分支     |
| **循环结构** | 在满足条件时重复执行某段代码       |

**共同特点**（⭐简答常考）：

- **只有一个入口**
- **只有一个出口**
- **结构内每一部分都有机会被执行到**
- **结构内不存在"死循环"**

> 由这三种基本结构组成的算法属于 **结构化算法**。

## 2.5 结构化程序设计方法（⭐简答常考）

**核心思想**：自顶向下、逐步细化、模块化设计、结构化编码

| 内容       | 说明                     |
| ---------- | ------------------------ |
| 自顶向下   | 从总体目标出发，逐层分解 |
| 逐步细化   | 将大问题逐步分解为小问题 |
| 模块化设计 | 将程序分为独立功能模块   |
| 结构化编码 | 仅用三种基本结构编写代码 |

## 3.2 数据类型（⭐核心必考）

```
数据类型
├── 基本类型
│   ├── 整型 int（2或4字节，-32768~32767 或 -2^31~2^31-1）
│   ├── 字符型 char（1字节，-128~127）
│   ├── 单精度浮点型 float（4字节，6~7位有效数字）
│   └── 双精度浮点型 double（8字节，15~16位有效数字）
├── 构造类型（数组、结构体、共用体、枚举）
├── 指针类型
└── 空类型 void
```

### ⚠️ 变长数组 (VLA) 应试避坑

```c
int n;
scanf("%d", &n);
int a[n];            // ❌ C99 支持，但期末考试极易被判语法错误！
```

> **考场铁律**：声明数组时 `[]` 内必须为**整型常量**。变量 `n` 作数组大小在笔试手写中通常扣分。

**两种标准替代方案**：

```c
// 方案1：预开辟足够大的常量数组
int a[1000];              // 题目说 n≤1000 就直接开满

// 方案2：动态内存分配（malloc/free，第8章详讲）
int *a = (int *)malloc(n * sizeof(int));
// ... 使用 ...
free(a);
```

## 3.6 本章错题复盘

### ⚠️ 3-4(2) char 溢出

**错误**：`char c1=197; printf("%d",c1)` 输出 197  
**正确**：197 超 signed char 范围(-128~127)，手算过程如下：

```
197 的 8 位二进制：     11000101
↓ 在 signed char 中，最高位为 1 → 判定为负数（计算机存的是补码）
补码 11000101 → 减1得反码 11000100
反码 11000100 → 符号位不变、其余取反 → 原码 10111011
原码 10111011 → 后7位 0111011 = 32+16+8+2+1 = 59
加上符号位：-59
```

> **⚡ 考场秒杀公式**：有符号 char 被赋予大于 127 的正数 X 时，直接用 **X - 256** 心算即得结果。  
> 例：`197 - 256 = -59`，`200 - 256 = -56`。无需纸面推导二进制补码，10秒出答案。

### ⚠️ 3-5 scanf %c 陷阱

**错误**：`a=3b=7 8.5 71.82 A a`（数字和字符间有空格）  
**正确**：`a=3b=7 8.5 71.82Aa`（字符紧跟数字后，%c 不跳空格）

## 3.7 高频考点

### 🔥 考点1：printf 格式符

- **题型**：读程序写结果
- `printf("%c %d", 97, 'a');` → 输出 `a 97`

### 🔥 考点2：scanf 取地址符

- **易错**：`scanf("%d", a);` ❌ → `scanf("%d", &a);` ✅

### 🔥 考点3：char 与 int 的关系

- char 存储的是 ASCII 码值（整数），char 和 int 在一定范围内可互换
- `'A' = 65`, `'a' = 97`, `'0' = 48`
- char 范围 -128~127，超过127会溢出

### 🔥 考点4：整数除法截断

- `5/3 = 1`（不是1.666）
- `1/2 = 0`（整数除法，结果为0）

### 🔥 考点5：++i 和 i++ 的区别

- 单独的 `i++;` 和 `++i;` 等价
- 参与表达式时不同：`a = i++` vs `a = ++i`

| 取余用浮点数           | `5.0 % 2`            | `%` 只能用于整数                      |
| 整数除整数             | `5/3` 得 1           | `5.0/3` 才得 1.666...                 |
| 字符常量/字符串混淆    | `"A"` 当字符用       | 字符用 `'A'`，字符串用 `"A"`          |
| scanf 格式多余字符     | `scanf("a=%d", &a)`  | 输入时必须原样输入 `a=3`              |
| char 溢出              | `c1=197` char 存不下 | char 范围 -128~127，改用 int          |
| printf 打印 char 用 %d | 把字符当整数输出     | 输出 ASCII 码用 `%d`，输出字符用 `%c` |

# 第4章 选择结构程序设计

## 4.1 关系运算符与关系表达式

| 运算符            | 含义                           | 优先级 |
| ----------------- | ------------------------------ | ------ |
| `<` `<=` `>` `>=` | 小于、小于等于、大于、大于等于 | 高     |
| `==` `!=`         | 等于、不等于                   | 低     |

- 关系表达式的结果：**真=1，假=0**
- **易错**：`==` 是等于，`=` 是赋值，千万别写错！

## 4.2 逻辑运算符与逻辑表达式（⭐必考）

| 运算符 | 含义          | 优先级 | 结合性 |
| ------ | ------------- | ------ | ------ |
| `!`    | 逻辑非（NOT） | 最高   | 右结合 |
| `&&`   | 逻辑与（AND） | 中     | 左结合 |
| `\|\|` | 逻辑或（OR）  | 最低   | 左结合 |

**优先级总排序**：单目（含 `!` `++` `--`等）> 算术 > 关系 > `&&` > `||` > 赋值

> ⚠️ 逻辑非 `!` 属于**单目运算符**，优先级高于所有算术运算符。`!a + b` 等价于 `(!a) + b`，而非 `!(a + b)`。

**短路特性**（⭐高频）：
- `a && b`：若 a 为假，不再计算 b
- `a || b`：若 a 为真，不再计算 b

### 🔥 短路求值 vs 运算符优先级：短路大过天

> **核心结论**：`&&` 和 `||` 的**从左往右短路机制**在运行时压倒一切。

```c
int a = 0, b = 0;
a && ++b;       // a=0 触发短路，++b 直接被"冷冻"，b 仍为 0
1 || printf("Hi"); // 左边为 1（真），短路！printf 永不执行
```

**无论右边表达式包含了什么**——`++`、`--`、函数调用、赋值，甚至套了三层括号——只要 `&&` 左边为假或 `||` 左边为真，**右边整个语法结构被完全无视**。这不是"优先级"的范畴，是 C 语言标准硬规定的执行流跳转。

- `a && b`：若 a 为假，不再计算 b
- `a || b`：若 a 为真，不再计算 b

**真假的判断**：C语言中，**非0即真，0为假**

## 4.3 if 语句（⭐核心）

### 三种形式

```c
// ① 单分支
if (条件) 语句;

// ② 双分支
if (条件) 语句1;
else     语句2;

// ③ 多分支
if (条件1)      语句1;
else if (条件2) 语句2;
else if (条件3) 语句3;
else            语句n;
```

### if-else 配对规则（⭐坑点）

**else 总是与前面最近的未配对的 if 配对！**

```c
if (a > 0)
    if (b > 0)
        c = 1;
    else          // ← 这个 else 和哪个 if 配对？
        c = -1;   // 答案：和 if(b>0) 配对，而非 if(a>0)
```

> 加 `{}` 可以改变配对关系，也是好习惯。

## 4.4 switch 语句

```c
switch (整型表达式)   // ⚠️ 只能是整型/字符型，不能是浮点型
{
    case 常量1: 语句1; break;
    case 常量2: 语句2; break;
    default:    语句n; break;
}
```

| 要点          | 说明                                        |
| ------------- | ------------------------------------------- |
| switch 表达式 | 必须为**整型或字符型**，不能是 float/double |
| case 后       | 必须为**常量表达式**，不能是变量            |
| break         | 跳出 switch，若漏写会**穿透**到下一个 case  |
| default       | 可放在任意位置，习惯放最后                  |

### ⚠️ default 位置与穿透陷阱

```c
// ❌ 风险写法：default 在最上方且无 break
switch (n) {
    default: printf("其他");    // 漏写 break！
    case 1:   printf("一");     // ← default 穿透下来执行了这行
}

// ✅ 安全写法：default 在最后一行（教材推荐规范）
switch (n) {
    case 1:   printf("一"); break;
    default:  printf("其他");   // 下方直接是 }，即使漏 break 也无物可穿透
}
```

> **核心辨析**：default 在最上方会向下穿透执行 case；default 在最下方直接"出门"（遇到 `}`），即使忘写 break 也不会误执行任何 case。考试读程序题常考此区别。

## 4.5 条件运算符（三目）

```c
max = (a > b) ? a : b;    // 若 a>b 为真取 a，否则取 b
//    表达式1 ? 表达式2 : 表达式3
```

- 结合性：**右结合**
- 优先级：仅高于赋值和逗号运算符

# 第5章 循环结构程序设计

## 5.1 三种循环语句（⭐核心必考）

### while 循环（先判断，后执行）

```c
while (条件表达式)
{
    循环体;
}
// 可能一次都不执行
```

### do...while 循环（先执行，后判断）

```c
do
{
    循环体;
} while (条件表达式);
// ⚠️ 分号不能丢！至少执行一次
```

### for 循环（最常用）

```c
for (表达式1; 表达式2; 表达式3)
{
    循环体;
}
// 表达式1：初始 → 表达式2：条件 → 循环体 → 表达式3：步进
```

| 对比         | while        | do-while     | for          |
| ------------ | ------------ | ------------ | ------------ |
| 最少执行次数 | 0次          | 1次          | 0次          |
| 适用场景     | 未知循环次数 | 至少执行一次 | 已知循环次数 |

### 🔥 do-while 选型判别口诀

> **"如果一开始条件就不满足，这块代码到底需不需要无条件执行一次？"**  
> 需要 → `do-while`；不需要 → `while`。

**do-while 三大高频应用场景**：

```c
// ① 密码/数据输入合法性检查（先输再判）
do {
    scanf("%d", &x);
} while (x < 0 || x > 100);

// ② 控制台菜单首次展示（先看再选）
do {
    printf("1.录入 2.查询 3.退出\n");
    scanf("%d", &choice);
} while (choice != 3);

// ③ 数字拆分兼容输入为 0（0 是一位数）
do {
    cnt++;           // 输入 0 → cnt=1，正确！
    n /= 10;
} while (n != 0);   // while 版本输入0会输出0位数（错误）
```

### 经典用法对比

```c
// while — 辗转相除法（未知迭代次数）
while (b != 0) { t = a % b; a = b; b = t; }

// do-while — 牛顿迭代法（至少迭代一次）
do { x0 = x1; x1 = (x0 + a/x0) / 2; } while (fabs(x1 - x0) > 1e-5);

// for — 阶乘累加（已知20次）
for (i = 1; i <= 20; i++) { temp *= i; sum += temp; }
```

## 5.2 循环控制语句

| 语句       | 作用                             | 关键点     |
| ---------- | -------------------------------- | ---------- |
| `break`    | 跳出**当前**循环                 | 只跳出一层 |
| `continue` | 跳过本次循环剩余部分，进入下一轮 | 不跳出循环 |

## 5.3 循环嵌套

- 外层循环一次，内层循环全部执行完
- 内层 break 只跳出内层，不影响外层

## 6.3 字符数组与字符串

```c
char s[] = "hello";          // 自动加\0，长度6
```

### 🔥 字符数组 ≠ 字符串：\0 是分水岭

```c
char a[] = {'h','e','l','l','o'};     // ❌ 纯字符数组，无 \0！
```

> 把没有 `\0` 的纯字符数组丢给 `%s` 输出 → **越界读内存**，屏幕上出现乱码或程序崩溃，考试读程序题常要求写出"输出乱码/非预期字符"。

### 三种输入对比

| 方法               | 遇空格 | 安全    | 换行处理   |
| ------------------ | ------ | ------- | ---------- |
| `scanf("%s")`      | 停     | ❌      | 残留缓冲区 |
| `gets()`           | 不停   | ❌ 危险 | 丢弃\n     |
| `fgets(s,n,stdin)` | 不停   | ✅ 安全 | **保留\n** |

| ------ | ------------------------------------- |

## 6.8 易错清单

1. 数组下标越界不报错！`a[10]` 访问的是第11个位置
2. `scanf("%s")` 遇空格就停，读不了带空格的字符串
3. 字符串必须 `\0` 结尾，忘了写会输出乱码
4. `gets()` 不安全但有考题，注意它丢弃 `\n`
5. `fgets()` 保留 `\n`，需要时手动去掉
6. 二维数组作参数：`int a[][N]` 第二维必须指定。原因是 C 语言采用**行优先连续存储**，编译器必须知道列数 N 才能通过公式 `地址 = 首地址 + (i × N + j) × sizeof(元素类型)` 计算出第 i 行第 j 列的偏移量，否则无法完成二维寻址
7. 折半查找要求数组已排序，升序降序的方向不同
8. 鞍点：同行可能有多个相同最大值，只取第一个可能漏检

# 第8章 善于利用指针

## 8.1 指针即地址 — 彻底理解 & 和 *

指针是 C 语言最强大的特性，也是最容易出错的地方。先搞清楚两个核心运算符：

| 运算符 | 名称 | 作用 | 示例 |
|--------|------|------|------|
| `&` | 取地址 | 获取变量的内存地址 | `&a` → a 的地址 |
| `*` | 解引用 | 获取指针指向的值 | `*p` → p 指向的那个变量的值 |

### 从内存视角理解指针

```c
int a = 10;          // 假设 a 存放在地址 0x1000
int *p = &a;         // p 的值就是 0x1000（存的是地址）

// 看图理解：
//   变量a                 变量p
//  ┌──────────┐         ┌──────────┐
//  │  10      │ ←────── │ 0x1000   │
//  │ 0x1000   │         │ 0x2000   │
//  └──────────┘         └──────────┘

printf("%d", a);     // 输出 10（直接拿 a 的值）
printf("%p", &a);    // 输出 0x1000（a 的地址）
printf("%d", *p);    // 输出 10（顺着 p 找到 a，取出值）
printf("%p", p);     // 输出 0x1000（p 存的就是 a 的地址）
printf("%p", &p);    // 输出 0x2000（p 自己的地址）
```

### `*` 的两种身份（⭐最容易混淆）

```c
int *p;        // ① 定义时：* 表示"p 是指针类型"，不是解引用
*p = 20;       // ② 执行时：* 表示"解引用"，往 p 指向的位置写值

// 两个 * 完全不同！一个是类型声明，一个是运算符。
```

### `&` 和 `*` 互为逆运算

```c
int a = 10;
int *p = &a;      // & 取地址赋给指针
*p = 20;          // * 解引用修改原变量
printf("%d", a);  // 输出 20 — a 被 *p 改掉了
```

## 8.2 指针类型速查

| 写法 | 含义 | 怎么读 |
|------|------|--------|
| `int *p` | 指向 int 的指针 | p 是 int 的指针 |
| `int **pp` | 指向"int指针"的指针（二级） | pp 是 int** 的指针 |
| `int (*p)[N]` | 指向数组的指针（数组指针） | p 指向含 N 个 int 的数组 |
| `int *p[N]` | 指针数组（N个指针） | p 是数组，元素是 int* |
| `double (*f)(double)` | 函数指针 | f 指向一个函数 |

### 🔥 指针类型秒杀：右左法则（括号优先）

```c
int *p;              // 名右边什么都没有 → 左边有* → 是指针
int **pp;            // 同上，多一层间接
int (*p)[N];         // 括号把 *p 绑一起 → p 是指针，右边是[] → 指向数组
int *p[N];           // 没括号 → p 右边是 [] → 是数组，左边是 int* → 元素是指针
double (*f)(double); // 括号把 *f 绑一起 → f 是指针，右边是() → 指向函数
```

> **秒杀口诀**：
> - 有圆括号强行将 `*` 与名字括起来 → **是指针**（指向数组 / 指向函数）
> - 没有圆括号，名字右边直接贴着 `[]` → **是数组**（指针数组）
> - 名字右边直接贴着 `(参数)` → **是函数**，返回指针

### 对比辨析：数组指针 vs 指针数组（⭐高频混淆）

```c
int (*p)[5];    // 数组指针 — p 指向"一个含 5 个 int 的数组"
                // p+1 跳过 5×4=20 字节（整个数组）

int *p[5];      // 指针数组 — p 是数组，有 5 个元素，每个是 int*
                // p[0], p[1], ... 各是一个指针，可以指向不同的地方
```

**实战场景区分**：
- 需要传二维数组给函数 → `int (*p)[N]`
- 需要对多个不等长字符串排序 → `char *p[N]`

## 8.3 指针运算 — 不是简单的加减

### 指针 ± 整数：跳过的是"所指类型的大小"

```c
int a[] = {10, 20, 30, 40, 50};
int *p = a;          // p 指向 a[0]

p++;                 // p 现在指向 a[1]，地址实际 +4（sizeof(int)）
// 不是地址 +1，是 +sizeof(int) = 4 字节！

// 步长自动适应类型：
char  *cp;  cp++;    // 地址 +1（sizeof(char)=1）
int   *ip;  ip++;    // 地址 +4（sizeof(int)=4）
double *dp; dp++;    // 地址 +8（sizeof(double)=8）
```

### 四种等价写法访问数组元素

```c
a[i]   ==   *(a + i)   ==   p[i]   ==   *(p + i)
// 以上四行完全等价，任意混用均可

// 示例：打印数组所有元素
int a[] = {1,2,3,4,5};
int *p = a;
for (int i = 0; i < 5; i++) {
    printf("%d ", a[i]);      // 最常见写法
    printf("%d ", *(a+i));    // 指针偏移
    printf("%d ", p[i]);      // 指针当下标用
    printf("%d ", *(p+i));    // 纯指针
}
```

### p++ vs (*p)++ vs *p++（⭐考试高频）

```c
int a = 10;
int *p = &a;

p++;      // 指针自增，指向下一个地址（危险！可能指向非法内存）
(*p)++;   // 先解引用拿到 a，再让 a 的值 +1 → a 变成 11
*p++;     // ⚠️ 陷阱！由于 * 和 ++ 同优先级、右结合：
          // 等价于 *(p++)，即先取 *p，再让 p 自增（不是 a 自增！）
```

### 指针相减：得到的是元素个数

```c
int a[] = {10, 20, 30, 40, 50};
int *p1 = &a[1];     // 指向 20
int *p2 = &a[4];     // 指向 50
printf("%d", p2 - p1); // 输出 3（不是 12 字节，是 3 个 int）
```

### 二维数组的指针运算

```c
int a[3][4];
// a 是 int (*)[4] 类型 — 指向含 4 个 int 的数组的指针

a + 1;        // 跳过一整行（4个int = 16字节），指向第 1 行
*(a + 1);     // 解引用得到第 1 行（仍然是地址，但类型变为 int*）
*(a + 1) + 2; // 第 1 行第 2 列的地址
*(*(a + 1) + 2); // 第 1 行第 2 列的值 ≡ a[1][2]
```

## 8.4 指针与函数 — 让函数"真正"修改外部变量

### 值传递 vs 地址传递（图解）

```c
// ❌ 值传递 — 交换失败
void swap_val(int x, int y) {
    int t = x; x = y; y = t;  // 只在函数内部换了副本，外面不变
}
// 调用：swap_val(a, b) → a 和 b 不变！

// ✅ 地址传递 — 交换成功
void swap_ptr(int *x, int *y) {
    int t = *x;    // t = 原x指向的值
    *x = *y;       // 把 y 指向的值写进 x 指向的位置
    *y = t;        // y 指向的位置写入 t
}
// 调用：swap_ptr(&a, &b) → a 和 b 真的换了！
```

> **本质**：C 语言所有参数传递都是"值传递"。地址传递本质是**把地址复制了一份传进去**，但通过这个地址可以间接修改原变量。

### 一维数组作参数

```c
// 以下四种声明完全等价，函数内部用法也一样
void f(int arr[], int n);
void f(int arr[10], int n);   // 10 被忽略，只当 int* 用
void f(int *arr, int n);
void f(int *arr, int n);      // 最坦诚的写法

// 因为数组名就是首元素地址，传数组 = 传指针
// 所以函数内修改 arr[i] 会直接影响原数组！
```

### 二维数组作参数 — 第二维必须写！

```c
// ✅ 正确写法 — 第二维必须指定
void f(int a[][4], int rows);     // int a[][4] 等价于 int (*a)[4]
void f(int (*a)[4], int rows);    // 指针形式，完全等价

// ❌ 错误写法 — 编译器无法计算偏移
void f(int a[][], int rows);      // 编译错误！
void f(int **a, int rows);        // 类型不匹配！二维数组 ≠ 二级指针
```

### 让函数返回指针（⚠️有坑）

```c
// ❌ 错误 — 返回局部变量地址
int *bad() {
    int x = 10;
    return &x;     // x 在函数返回后已被销毁，返回的地址指向垃圾！
}

// ✅ 正确做法 1 — 返回静态变量地址
int *ok1() {
    static int x = 10;   // static 变量不随函数返回销毁
    return &x;
}

// ✅ 正确做法 2 — 返回 malloc 的地址（调用者负责 free）
int *ok2() {
    int *p = (int *)malloc(sizeof(int));
    *p = 10;
    return p;            // malloc 的空间在堆上，不随函数销毁
}

// ✅ 正确做法 3 — 返回传入的指针（指针作参数）
int *ok3(int *p) {
    *p = 10;
    return p;            // 返回的还是调用者传进来的地址
}
```

## 8.5 动态内存 — 按需分配与释放

### malloc：在堆上分配内存

```c
#include <stdlib.h>     // malloc, free, sizeof

int n;
scanf("%d", &n);

// 动态分配 n 个 int 的空间
int *a = (int *)malloc(n * sizeof(int));
if (a == NULL) {         // ⚠️ 必须检查分配是否成功
    printf("内存不足！\n");
    return -1;
}

// 使用方式完全等同于数组
for (int i = 0; i < n; i++)
    a[i] = i * 2;        // 和数组一模一样！

free(a);                 // ⚠️ 用完必须释放！否则内存泄漏
a = NULL;                // 好习惯：释放后置空，防止野指针
```

### malloc / calloc / realloc 对比

| 函数 | 作用 | 初始化 | 参数 |
|------|------|--------|------|
| `malloc(n)` | 分配 n 字节 | 不初始化（随机值） | `malloc(n * sizeof(int))` |
| `calloc(cnt, size)` | 分配 cnt×size 字节 | **全部初始化为 0** | `calloc(n, sizeof(int))` |
| `realloc(p, n)` | 调整已分配块的大小 | 保留原内容 | `realloc(p, n * sizeof(int))` |
| `free(p)` | 释放内存 | — | `free(p); p = NULL;` |

```c
// calloc 示例 — 适合"分配即清零"的场景
int *a = (int *)calloc(n, sizeof(int));  // 所有元素初始为 0

// realloc 示例 — 扩容
int *tmp = (int *)realloc(a, n * 2 * sizeof(int));
if (tmp) a = tmp;  // 成功则更新指针，失败则原内存仍有效
```

### 常见内存错误

```c
// ❌ 1. 忘记 free → 内存泄漏
int *p = malloc(100);  // 用完后没 free，程序运行越久内存越少

// ❌ 2. 释放后继续用 → 野指针
free(p);
*p = 10;              // 非法！p 已失效

// ❌ 3. 重复释放
free(p);
free(p);              // 非法！程序可能崩溃

// ❌ 4. free 非 malloc 返回的指针
int a[10];
free(a);              // 非法！a 是栈上的数组，不是 malloc 来的
```

## 8.6 指针经典用法 — 逐类详解

### 用法一：用指针遍历字符串

```c
// 写法1：下标法（入门）
char str[] = "Hello";
for (int i = 0; str[i] != '\0'; i++)
    putchar(str[i]);

// 写法2：指针法（高手）
char *p = str;
while (*p != '\0') {     // 等价于 while (*p)
    putchar(*p);
    p++;
}

// 写法3：极致精简版
for (char *p = str; *p; p++)
    putchar(*p);
```

### 用法二：交换两个变量（指针作参数）

```c
void swap(int *a, int *b) {
    int t = *a;
    *a = *b;
    *b = t;
}
// 调用：swap(&x, &y);
```

### 用法三：指针数组排序 — 只换指向不换内容

```c
// 场景：10 个不等长字符串排序
char *str[10];                       // 指针数组：10 个字符串指针
for (int i = 0; i < 10; i++) {
    char buf[100];
    scanf("%s", buf);
    str[i] = (char *)malloc(strlen(buf) + 1);  // 各自分配恰好够的空间
    strcpy(str[i], buf);
}

// 冒泡排序 — 只交换指针，不动字符串内容
for (int i = 0; i < 9; i++)
    for (int j = 0; j < 9 - i; j++)
        if (strcmp(str[j], str[j+1]) > 0) {
            char *t = str[j];        // 只改指针指向！
            str[j] = str[j+1];
            str[j+1] = t;
        }
// 字符串本体从未移动，只是 str[0], str[1]... 指的方向变了

for (int i = 0; i < 10; i++)
    free(str[i]);                    // 别忘了释放
```

### 用法四：二级指针 — 指针的指针

```c
int a = 10;
int *p = &a;       // p → a
int **pp = &p;     // pp → p → a

printf("%d", a);   // 10
printf("%d", *p);  // 10  — 解引用一次
printf("%d", **pp);// 10  — 解引用两次

// 二级指针常见用途：在函数中修改指针本身
void make_null(int **pp) {
    *pp = NULL;     // 让外面的指针变为 NULL
}
// 调用：int *p = &a;  make_null(&p);  → p 变成 NULL
```

**二级指针排序整数**：

```c
void sort(int **arr, int n) {        // arr 是指针数组
    for (int i = 0; i < n - 1; i++)
        for (int j = 0; j < n - 1 - i; j++)
            if (**(arr + j) > **(arr + j + 1)) {  // 比较值
                int *t = *(arr + j);              // 交换指针
                *(arr + j) = *(arr + j + 1);
                *(arr + j + 1) = t;
            }
}
// 核心：比较用 **(arr+j) 取到整数，交换用 *(arr+j) 即指针
```

### 用法五：函数指针 — 把函数当参数传

```c
// 通用积分函数 — 对任意函数 f 在 [a,b] 上做定积分
double integral(double (*f)(double), double a, double b, int n) {
    double h = (b - a) / n;
    double sum = 0.0;
    for (int i = 0; i < n; i++)
        sum += f(a + i * h) * h;     // 调用传入的函数
    return sum;
}

// 调用时直接传函数名：
integral(sin,  0, 1, 100000);   // ∫₀¹ sin(x)dx
integral(cos,  0, 1, 100000);   // ∫₀¹ cos(x)dx
integral(exp,  0, 1, 100000);   // ∫₀¹ eˣdx
integral(sqrt, 0, 1, 100000);   // ∫₀¹ √x dx

// 函数指针的本质：一个变量，指向某段代码的入口地址
// 函数名本身就是地址，等同于 &函数名
```

## 8.9 指针经典用法速查

| 题型 | 形参写法 | 核心技巧 |
|------|---------|---------|
| 变量交换 | `int *p1, *p2` | swap 解引用交换 `*p1` 和 `*p2` |
| 一维数组遍历 | `int *arr` | `*(arr+i)` ≡ `arr[i]`；`p++` 自动跳过 sizeof |

| 动态内存 | `malloc` / `free` | 用完必须 `free`；free 后 `p=NULL` |

## 8.10 易错清单

1. 野指针：未初始化就解引用 → 程序崩溃 `int *p; *p=10;` ❌
2. `scanf("%d", p)` 不需要 `&`（p 本身就是地址，`&p` 是指针的地址）
3. `int *p` 的 `*`（类型声明）和 `*p` 的 `*`（解引用运算符）完全不同
4. `*(a + i)` ≠ `*a + i`：前者 = `a[i]`，后者 = `a[0] + i`
5. 指针数组排序只交换指针指向，不移动字符串内容
6. `malloc` 后必须 `free`；`free` 后立即置 `NULL` 防野指针
7. 数组名是常量指针：`a = p` ❌；`p = a` ✅
8. 不能返回局部变量的地址（函数返回后局部变量已销毁）
9. 二维数组 `int a[3][4]` 的类型是 `int (*)[4]`，不是 `int **`
10. `p++` 是移动指针，`(*p)++` 是修改指向的值，`*p++` 等价 `*(p++)`

## A.5 scanf 四大陷阱（⭐考试最高频）

### 陷阱一：%c 不跳过空白字符

```c
scanf("%d", &n);
scanf("%c", &ch);
// 输入：123换行A
// 结果：n=123, ch='\n' （读到了换行符，不是'A'！）
```

**解决方案**：

```c
// 方案1：在 %c 前加空格，让 scanf 跳过所有空白
scanf(" %c", &ch);  // 注意 %c 前面有个空格！

// 方案2：用 getchar() 吃掉缓冲区残留
scanf("%d", &n);
getchar();          // 吃掉换行符
scanf("%c", &ch);
```

### 陷阱二：格式串中的普通字符必须原样输入

```c
scanf("x=%f,y=%f", &x, &y);
// 必须输入：x=8.5,y=71.82  ← 字母和逗号一个不能少
```

### 陷阱三：scanf 到 %s 不读空格

```c
char str[20];
scanf("%s", str);
// 输入：Hello World
// 结果：str = "Hello" （World 留在缓冲区）
```

### 陷阱四：浮点类型混淆

```c
float  x;   scanf("%f", &x);   // 正确：用 %f
double y;   scanf("%lf", &y);  // 正确：用 %lf  ← 区分！

// printf 输出：现代 C 标准中 printf 打印 double 可用 %f 或 %lf
// 但为考试绝对安全且兼容老旧编译器，建议牢记：
//   输入（scanf）必须区分：float 用 %f，double 用 %lf
//   输出（printf）一律用 %f（float 和 double 都用 %f）
printf("%f", x);   // 正确
printf("%f", y);   // 正确（printf 对所有浮点类型一律用 %f）
```

| 格式串有普通字符 | 只输入数据               | 必须原样输入            |

## A.9 期末考试常见题型

### 题型一：写输出结果

```c
int a = 5, b = 7;
printf("a=%d,b=%d\n", a, b);
// 输出：a=5,b=7
```

### 题型二：字符与ASCII互转

```c
char c = 'A';
printf("%c %d\n", c, c);
// 输出：A 65
```

### 题型三：格式控制

```c
printf("%5d\n", 123);    // 输出：  123
printf("%05d\n", 123);   // 输出：00123
printf("%.2f\n", 3.1);   // 输出：3.10
printf("%-5d\n", 123);   // 输出：123
```

### 题型四：scanf 陷阱分析

```c
int n; char c;
scanf("%d", &n);
scanf("%c", &c);
// 输入：10换行A
// 问：c 读到了什么？
// 答：'\n'（换行符），不是 'A'
```

### 题型五：格式串中的普通字符

```c
scanf("a=%db=%d", &a, &b);
// 应如何输入使 a=3, b=7？
// 答：a=3b=7
```

### 题型六：char 溢出

```c
char c = 197;           // 超出 signed char 范围
printf("%d\n", c);
// 输出：-59 （发生溢出）
```

### 题型七：短路求值

```c
int a = 3, b = 4;
!(a > b) && !(c = 5) && 0;
// 问：c 的值？
// 答：c 未被赋值，因为 && 短路 — 左边为假，右边不执行
```
