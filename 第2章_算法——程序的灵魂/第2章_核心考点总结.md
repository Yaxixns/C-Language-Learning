# 第2章 算法——程序的灵魂 — 核心考点总结

---

## 一、本章核心知识点

### 1.1 算法的概念（⭐必考）

| 概念 | 定义 |
|------|------|
| **算法** | 为解决一个问题而采取的方法和步骤，是程序的灵魂 |
| **程序** | 数据结构 + 算法 + 程序设计方法 + 语言工具 |

**通俗理解**：算法 = 做事的步骤。比如菜谱、洗衣机流程、乐谱等。

### 1.2 算法的五大特性（⭐高频简答）
1. **有穷性**：有限步骤后结束，每一步在有限时间内完成
2. **确定性**：每一步都有确切的含义，不能有二义性
3. **有零个或多个输入**：从外界获取必要信息
4. **有一个或多个输出**：必须产生结果
5. **有效性**：每一步都能被有效执行，能得到确定结果

### 1.3 算法的表示方法（⭐常考选择题）

| 方法 | 说明 | 特点 |
|------|------|------|
| 自然语言 | 用日常语言描述 | 通俗但易歧义 |
| **传统流程图** | 用图形符号表示算法 | 直观形象，但占用篇幅大 |
| **N-S 流程图** | 盒图，去掉流程线 | 结构化，紧凑 |
| **伪代码** | 介于自然语言和代码之间 | 灵活，接近程序 |
| 计算机语言 | 用 C 等语言实现 | 计算机可直接执行 |

**流程图常用符号**：
| 符号 | 名称 | 含义 |
|------|------|------|
| ⬭ 圆角矩形 | 起止框 | 算法开始/结束 |
| ⬜ 矩形 | 处理框 | 计算/赋值 |
| ◇ 菱形 | 判断框 | 条件判断 |
| ➡ 箭头 | 流程线 | 执行方向 |
| ▱ 平行四边形 | 输入/输出框 | 数据读写 |

### 1.4 三种基本结构（⭐核心必考）

```
顺序结构          选择结构              循环结构
   A                |                    ┌───┐
   |             ┌──◇──┐                │条件│
   B             │T   F│                └─◇─┘
   |             A     B                T│  │F
   C             |     |                A  ╳
```

| 结构 | 特点 |
|------|------|
| **顺序结构** | 按先后顺序依次执行，无分支、无跳转 |
| **选择结构** | 根据条件判断，选择执行某个分支 |
| **循环结构** | 在满足条件时重复执行某段代码 |

**共同特点**（⭐简答常考）：
- **只有一个入口**
- **只有一个出口**
- **结构内每一部分都有机会被执行到**
- **结构内不存在"死循环"**

> 由这三种基本结构组成的算法属于 **结构化算法**。

### 1.5 结构化程序设计方法（⭐简答常考）

**核心思想**：自顶向下、逐步细化、模块化设计、结构化编码

| 内容 | 说明 |
|------|------|
| 自顶向下 | 从总体目标出发，逐层分解 |
| 逐步细化 | 将大问题逐步分解为小问题 |
| 模块化设计 | 将程序分为独立功能模块 |
| 结构化编码 | 仅用三种基本结构编写代码 |

## 三、易错点警示（⚠️ 坑点汇总）

| 易错点 | 说明 |
|--------|------|
| 算法 ≠ 程序 | 算法是思路，程序是算法的具体实现 |
| 有穷性 ≠ 死循环 | 循环必须有退出条件 |
| 基本结构特点 | 必须"单入口单出口"，自己设计的基本结构也必须满足 |
| 流程图菱形框 | 必须有两个出口（T/F），不能只有一个 |
| N-S 图 vs 传统流程图 | N-S 图无流程线，传统流程图有箭头和流向线 |
