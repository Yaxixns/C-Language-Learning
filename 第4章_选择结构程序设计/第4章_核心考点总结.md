# 第4章 选择结构程序设计 — 核心考点总结

---

## 一、本章核心知识点

### 1.1 关系运算符与关系表达式

| 运算符 | 含义 | 优先级 |
|--------|------|--------|
| `<` `<=` `>` `>=` | 小于、小于等于、大于、大于等于 | 高 |
| `==` `!=` | 等于、不等于 | 低 |

- 关系表达式的结果：**真=1，假=0**
- **易错**：`==` 是等于，`=` 是赋值，千万别写错！

### 1.2 逻辑运算符与逻辑表达式（⭐必考）

| 运算符 | 含义 | 优先级 | 结合性 |
|--------|------|--------|--------|
| `!` | 逻辑非（NOT） | 最高 | 右结合 |
| `&&` | 逻辑与（AND） | 中 | 左结合 |
| `\|\|` | 逻辑或（OR） | 最低 | 左结合 |

**优先级总排序**：`!` > 算术 > 关系 > `&&` > `||`

**短路特性**（⭐高频）：
- `a && b`：若 a 为假，不再计算 b
- `a || b`：若 a 为真，不再计算 b

**真假的判断**：C语言中，**非0即真，0为假**

### 1.3 if 语句（⭐核心）

#### 三种形式
```c
// ① 单分支
if (条件) 语句;

// ② 双分支
if (条件) 语句1;
else     语句2;

// ③ 多分支
if (条件1)      语句1;
else if (条件2) 语句2;
else if (条件3) 语句3;
else            语句n;
```

#### if-else 配对规则（⭐坑点）
**else 总是与前面最近的未配对的 if 配对！**

```c
if (a > 0)
    if (b > 0)
        c = 1;
    else          // ← 这个 else 和哪个 if 配对？
        c = -1;   // 答案：和 if(b>0) 配对，而非 if(a>0)
```

> 加 `{}` 可以改变配对关系，也是好习惯。

### 1.4 switch 语句

```c
switch (整型表达式)   // ⚠️ 只能是整型/字符型，不能是浮点型
{
    case 常量1: 语句1; break;
    case 常量2: 语句2; break;
    default:    语句n; break;
}
```

| 要点 | 说明 |
|------|------|
| switch 表达式 | 必须为**整型或字符型**，不能是 float/double |
| case 后 | 必须为**常量表达式**，不能是变量 |
| break | 跳出 switch，若漏写会**穿透**到下一个 case |
| default | 可放在任意位置，习惯放最后 |

### 1.5 条件运算符（三目）

```c
max = (a > b) ? a : b;    // 若 a>b 为真取 a，否则取 b
//    表达式1 ? 表达式2 : 表达式3
```

- 结合性：**右结合**
- 优先级：仅高于赋值和逗号运算符

## 二、高频考点（期末必考）

### 🔥 考点1：if-else 配对问题
程序读结果题必出，嵌套 if 的 else 归属判断。
> **口诀**：else 找最近没配对的 if

### 🔥 考点2：逻辑表达式求值 + 短路
- `a=3,b=4,c=5; a+b>c && b==c` → `7>5 && 4==5` → `1 && 0` → `0`
- `!(a>b) && !c || 1` → 注意 `|| 1` 直接为真

### 🔥 考点3：switch 漏写 break
- 漏 break 会导致"穿透"，连续执行多个 case

### 🔥 考点4：分段函数编程
- 核心题型：用 if-else 或 switch 实现分段函数
- printf/scanf 格式要与输入输出一致

## 三、易错点警示（⚠️ 坑点汇总）

| 易错点 | 错误 | 正确 |
|--------|------|------|
| = 与 == 混淆 | `if (a = 3)` 永远为真 | `if (a == 3)` |
| else 配对错 | 缩进误导 | else 找最近未配对的 if |
| switch 用浮点数 | `switch(2.5)` | switch 只能用整型/字符型 |
| case 后用变量 | `case a:` | case 只能用常量 |
| 漏写 break | 穿透执行 | 需要的时候必须写 break |
| 逻辑优先级 | `!a && b` 以为是 `!(a&&b)` | `!` 优先级高于 `&&` |
| 短路求值 | 误以为右侧必执行 | `&&` 左侧为假右侧不执行 |
